'''Zen C''' is a modern systems programming language that compiles to human-readable <code>GNU C</code>/<code>C11</code>. It provides a rich feature set including type inference, pattern matching, generics, traits, async/await, and manual memory management with RAII capabilities. The language achieves all of this while maintaining 100% C ABI compatibility. The core philosophy of Zen C is to "Write like a high-level language, run like C".

Zen C is designed to work with most C11 compilers, with full test suite passes on GCC, Clang, and Zig CC, as well as high support on TCC. It also features built-in interoperability flags that allow the language to transpile to C++ (<code>--cpp</code>), CUDA C++ for GPU programming (<code>--cuda</code>), and Objective-C (<code>--objc</code>).

==Overview==

===Variables and Constants===
Zen C distinguishes between compile-time constants and runtime variables.
* Manifest constants are declared using the <code>def</code> keyword and exist only at compile-time (folded directly into the code).
* Runtime variables are declared using the <code>let</code> keyword and represent storage locations in memory.
* Zen C automatically infers the types for initialized variables.
* Depending on the backend compiler, this type inference compiles down to C23's <code>auto</code> or GCC's <code>__auto_type</code> extension.

<syntaxhighlight lang="rust">
def MAX_SIZE = 1024;   // Compile-time constant
let x = 10;            // Mutable variable with inferred type
let y: const int = 10; // Read-only variable with explicit type annotation
</syntaxhighlight>

===Functions and Lambdas===
Functions are declared using the <code>fn</code> keyword and define their return types with an arrow <code>-></code>. 
* Zen C supports named arguments during function calls, which must strictly follow the defined parameter order.
* Functions can define default values for trailing arguments, ranging from simple literals to complex struct constructors.
* The language supports anonymous functions (lambdas/closures) that can capture their environment by value or by reference using block or arrow syntax.

<syntaxhighlight lang="rust">
fn add(a: int, b: int) -> int {
    return a + b;
}

// Arrow syntax lambda
let factor = 2;
let doubler = x -> x * factor; 

// Block syntax lambda capturing by reference
let val = 10;
let modify = fn[&]() { val += 1; };
</syntaxhighlight>

===Control Flow===
Zen C offers robust control flow mechanisms beyond standard conditionals and loops.
* The <code>match</code> statement serves as a powerful alternative to <code>switch</code>, allowing for range matching and enum destructuring.
* Loops include exclusive and inclusive ranges (<code>for i in 0..10</code>), step increments, and direct iteration over arrays or types implementing the <code>Iterable</code> trait.

<syntaxhighlight lang="rust">
match val {
    1         => { print "One" },
    2 || 3    => { print "Two or Three" },
    10 ..= 25 => { print "10 to 25" },
    _         => { print "Other" },
}

// Destructuring an Enum
match shape {
    Shape::Circle(r)   => { println "Radius: {r}" },
    Shape::Rect(w, h)  => { println "Area: {w*h}" },
    Shape::Point       => { println "Point" },
}
</syntaxhighlight>

===Memory Management===
Zen C does not feature a garbage collector, favoring manual memory management combined with ergonomic features to prevent leaks.
* The <code>defer</code> statement is used to execute cleanup code when the current scope exits, executing in a last-in, first-out (LIFO) order.
* The <code>autofree</code> keyword automatically frees a variable upon scope exit.
* To prevent double-free errors, types with destructors are treated as "Resources" and strictly follow Move Semantics by default (ownership is transferred on assignment).
* RAII (Resource Acquisition Is Initialization) is natively supported by implementing the <code>Drop</code> trait.

<syntaxhighlight lang="rust">
let f = fopen("file.txt", "r");
defer fclose(f); // Guaranteed to run when the surrounding function/scope ends
</syntaxhighlight>

===Object-Oriented Programming===
Zen C allows for object-oriented design patterns through custom structures and methods.
* Methods are attached to structs (or even primitive types like <code>int</code>) using the <code>impl</code> block.
* Shared behavior and interfaces are defined using the <code>trait</code> keyword.
* Instead of traditional inheritance, Zen C uses composition via the <code>use</code> keyword, which allows developers to mix in or nest fields from other structs.

<syntaxhighlight lang="rust">
struct Point {
    x: int;
    y: int;
}

impl Point {
    // Constructor convention
    fn new(x: int, y: int) -> Self {
        return Point{x: x, y: y};
    }

    // Instance method using '.field' self shorthand
    fn dist(self) -> float {
        return sqrt(.x * .x + .y * .y); 
    }
}
</syntaxhighlight>

===Metaprogramming===
Zen C features powerful compile-time execution and code generation capabilities.
* Code enclosed in a <code>comptime</code> block runs during the compilation phase, allowing for dynamic source generation or custom compile-time errors.
* The <code>embed</code> keyword allows binary or text files to be embedded directly into the executable as specific types (e.g., strings, byte arrays).
* Attributes like <code>@cfg()</code> enable conditional compilation based on defined flags.

<syntaxhighlight lang="rust">
comptime {
    // Generates source code during the build process
    println "let build_date = \"2026-03-07\";";
}

println "Build Date: {build_date}";
</syntaxhighlight>
