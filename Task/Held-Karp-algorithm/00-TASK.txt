; Held–Karp Algorithm Problem Description

[https://en.wikipedia.org/wiki/Held%E2%80%93Karp_algorithm The Held–Karp algorithm] is a classic dynamic programming algorithm used to solve the Traveling Salesperson Problem (TSP). While the TSP is an NP-hard problem, the Held–Karp algorithm provides an exact solution in exponential time, which is considerably more efficient than a brute-force search for small to moderately sized inputs.

; Problem Statement: Traveling Salesperson Problem

Given a set of <math>n</math> cities and a matrix <math>C</math> where <math>C_{ij}</math> represents the cost (or distance) of traveling from city <math>i</math> to city <math>j</math>, the Traveling Salesperson Problem (TSP) requires finding the minimum-cost route that visits each city exactly once and returns to the starting city.

Let the cities be indexed from <math>0</math> to <math>n-1</math>. We seek a permutation of cities <math>(v_0, v_1, \dots, v_{n-1})</math>, where <math>v_0</math> is the starting city (e.g., city 0), such that the total cost of the cycle

<math>
\text{Total Cost} = \sum_{i=0}^{n-2} C_{v_i, v_{i+1}} + C_{v_{n-1}, v_0}
</math>


is minimized.

; Algorithm Overview

The Held–Karp algorithm employs dynamic programming to solve the TSP. It breaks down the problem into smaller subproblems defined by the set of cities visited so far and the current location of the salesperson.

; Dynamic Programming State

Let <math>dp(S, j)</math> represent the minimum cost of a path that starts at a designated city (typically city 0), visits all cities in the set <math>S</math> exactly once, and ends at city <math>j</math>, where <math>j \in S</math> and the starting city <math>0 \in S</math>.

Here, <math>S</math> is a subset of the set of all cities <math>V = {0, 1, \dots, n-1}</math>, and <math>j</math> is a city in the subset <math>S</math>. Note that we fix the starting city as 0 and require <math>0 \in S</math> for any state.

; Base Case

The base case for the dynamic programming is a path starting at city 0 and going directly to another city <math>j</math>.
For all cities <math>j \in {1, 2, \dots, n-1}</math>:

<math>
dp(\{0, j\}, j) = C_{0, j}
</math>


This represents the shortest path from city 0 visiting only cities 0 and <math>j</math>, ending at city <math>j</math>.

; Recurrence Relation

To compute <math>dp(S, j)</math> for a subset <math>S</math> with <math>|S| > 2</math> and <math>j \ne 0</math>, the path must have arrived at city <math>j</math> from some city <math>i</math> where <math>i \in S</math> and <math>i \ne j</math>. The path leading to city <math>i</math> must have visited all cities in the subset <math>S \setminus {j}</math> and ended at <math>i</math>. The cost of this path is <math>dp(S \setminus {j}, i)</math>. Therefore, the minimum cost to reach city <math>j</math> having visited all cities in <math>S</math> is the minimum cost over all possible preceding cities <math>i</math> plus the cost of the final leg from <math>i</math> to <math>j</math>.

For <math>S \subseteq V</math> with <math>|S| > 2</math>, <math>0 \in S</math>, and <math>j \in S, j \ne 0</math>:

<math>
dp(S, j) = \min_{i \in S, i \ne j} \{ dp(S \setminus \{j\}, i) + C_{i, j} \}
</math>


The algorithm computes these values iteratively, starting with subsets of size 2, then size 3, and so on, up to size <math>n</math>.

; Final Solution

The final goal is to find the shortest Hamiltonian cycle starting and ending at city 0. This means we need to find the shortest path that visits all cities in <math>V</math>, ends at some city <math>j \ne 0</math>, and then add the cost of returning from <math>j</math> to 0.

The minimum total cost for the TSP is:

<math>
\min_{j \in \{1, \dots, n-1\}} \{ dp(V, j) + C_{j, 0} \}
</math>


where <math>V</math> is the set of all <math>n</math> cities.

; Complexity

The time complexity of the Held–Karp algorithm is determined by the number of states and the cost to compute each state.

Number of states <math>(S, j)</math>: There are <math>2^{n-1}</math> possible subsets <math>S</math> that contain city 0. For each subset <math>S</math> with <math>k</math> cities, there are <math>k-1</math> possible ending cities <math>j \ne 0</math>. The total number of states is bounded by <math>O(n 2^n)</math>.

Transition cost: To compute <math>dp(S, j)</math>, we iterate through all possible previous cities <math>i \in S \setminus {j}</math>. There are at most <math>n-1</math> such cities.

Overall time complexity: <math>O(n \cdot 2^n \cdot n) = O(n^2 2^n)</math>.

The space complexity required to store the DP table <math>dp(S, j)</math> is proportional to the number of states, which is <math>O(n 2^n)</math>.

While exponential, this is significantly better than the <math>O(n!)</math> complexity of brute-force enumeration of all possible permutations.
