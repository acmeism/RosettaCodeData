;Task:
Display the current CPU utilization, as a percentage, calculated from <code> /proc/stat</code>.


;Background:
Most Linux kernels provide a virtual &nbsp; <code>[https://www.kernel.org/doc/Documentation/filesystems/proc.txt /proc]</code> &nbsp; filesystem, providing an interface to various internal data structures. &nbsp; 

One of these internal structures (<code>/proc/stat</code>) includes information on the amount of time (in <code>USER_HZ</code>) spent in various states. &nbsp; From this information, we can, with a little effort, determine the current level of CPU utilization, as a percent of time spent in any states other than ''idle''.

To do this:
:::# &nbsp; read the first line of &nbsp; <code>/proc/stat</code>
:::# &nbsp; discard the first word of that first line &nbsp; (it's always <code>cpu</code>)
:::# &nbsp; sum all of the times found on that first line to get the total time
:::# &nbsp; divide the fourth column ("idle") by the total time, to get the fraction of time spent being idle
:::# &nbsp; subtract the previous fraction from 1.0 to get the time spent being &nbsp; ''not'' &nbsp; idle
:::# &nbsp; multiple by &nbsp; '''100''' &nbsp; to get a percentage

<br>
The times in <code>/proc/stat</code> are monotonically increasing, and begin at some point during the kernel's initialization (ie during boot up). &nbsp; So the steps above will give the total CPU utilization since boot, which may or may not be useful to some. &nbsp; To get a more real-time utilization, we simply repeat the steps above with some small sleep interval in between, and instead of using the absolute total and idle times, we use use the total time delta and the idle time delta to compute the utilization.

This project is based on &nbsp; [http://colby.id.au/calculating-cpu-usage-from-proc-stat/ this blog post] &nbsp; by [[User:Paul|Paul Colby]], &nbsp; and the &nbsp; [[#UNIX Shell|Bash]] &nbsp; version comes from there.
