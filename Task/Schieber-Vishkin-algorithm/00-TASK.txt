The [https://gist.github.com/diegode/856193 '''Schieber-Vishkin algorithm'''] is a parallel algorithm for finding the '''lowest common ancestor (LCA)''' of two nodes in a tree. It operates in the '''concurrent-read exclusive-write (CREW) PRAM model''' and is notable for its efficiency, achieving logarithmic time complexity. The algorithm preprocesses the tree in <math>O(n)</math> time and allows subsequent LCA queries to be answered in <math>O(1)</math> time.

;Problem definition

The '''lowest common ancestor''' problem is defined as follows:

Given a rooted tree <math>T</math> with <math>n</math> nodes, and two nodes <math>u</math> and <math>v</math> in the tree, find the deepest node <math>w</math> that is an ancestor of both <math>u</math> and <math>v</math>. Formally, <math>w</math> satisfies:

<math>
w \in \text{Ancestors}(u) \cap \text{Ancestors}(v)
</math>

and for any other node <math>w'</math> in the intersection:

<math>
\text{depth}(w') \leq \text{depth}(w).
</math>

;Input

1. A rooted tree <math>T = (V, E)</math>, where <math>V</math> is the set of nodes and <math>E</math> is the set of edges. The tree has <math>n</math> nodes.
2. A pair of nodes <math>(u, v)</math>, where <math>u, v \in V</math>.

;Output

The algorithm outputs the node <math>w</math> that is the lowest common ancestor of <math>u</math> and <math>v</math>.

;Algorithm overview

The Schieber-Vishkin algorithm is designed to preprocess the tree in '''parallel''' to enable constant-time LCA queries. The preprocessing involves assigning each node a label that encodes information about its position in the tree hierarchy. These labels are constructed using tree structures such as parent-child relationships and depth.

The key steps of the algorithm are:

1. '''Tree representation:''' The input tree is represented as an adjacency list or similar structure. The root node is identified.
2. '''Preprocessing in parallel:''' During preprocessing:
   - Nodes are assigned labels based on their position in the tree.
   - Depth-first search (DFS) or similar traversal techniques are used to compute necessary information.
3. '''Query resolution:''' Using the precomputed labels, the LCA of any two nodes <math>u</math> and <math>v</math> can be determined in <math>O(1)</math> time.

;Complexity

- '''Preprocessing time:''' <math>O(n)</math> using <math>O(n / \log n)</math> processors in the CREW PRAM model.
- '''Query time:''' <math>O(1)</math>.
- '''Space complexity:''' <math>O(n)</math>.

;Applications

The Schieber-Vishkin algorithm is widely used in applications where fast query resolution is critical, such as:

- Network routing and communication.
- Computational biology, particularly in phylogenetics.
- Query optimization in databases.
- Text processing, such as suffix tree and suffix array construction.

This algorithm is a foundational result in '''parallel computing''' and demonstrates how preprocessing can enable efficient query answering in tree-based problems.
