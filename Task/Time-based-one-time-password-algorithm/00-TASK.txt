A Time-based One-time Password Algorithm (TOTP) is an algorithm that computes a one-time password from a shared secret key and the current time.  It is the cornerstone of [[wp:Initiative_For_Open_Authentication|Initiative For Open Authentication (OATH)]] and is used in a number of two factor authentication systems. 

Essentially, both the server and the client compute the time-limited token, then the server checks if the token supplied by the client matches the locally generated token.

A reference implementation, based on JavaScript, can be found at the following location:

According to RFC 6238, the reference implementation is as follows:
* &nbsp; Generate a key, '''K''', which is an arbitrary bytestring, and share it securely with the client.
*&nbsp;  Agree upon an epoch, T0, and an interval, TI, which will be used to calculate the value of the counter C (defaults are the Unix epoch as T0 and 30 seconds as TI)
* &nbsp; Agree upon a cryptographic hash method (default is SHA-1)
* &nbsp; Agree upon a token length, '''N''' (default is 6)
	

Although RFC 6238 allows different parameters to be used, the Google implementation of the authenticator app does not support T0, TI values, hash methods and token lengths different from the default. &nbsp; It also expects the '''K''' secret key to be entered (or supplied in a QR code) in base-32 encoding according to RFC 3548.
