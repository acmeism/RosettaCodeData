Within a [[wp:String (computer science)|string]], an [[wp:Escape sequence|escape sequence]] gives special meaning to otherwise ordinary characters.

For example, in the [[wp:C (programming language)|C programming language]], the string literal <code>"a\nb"</code>, containing the escape sequence <code>\n</code>, would be interpreted by the compiler as the bytes <code>61 0a 62</code>, not <code>61 5c 6e 62</code>.

In [[wp:JSON|JavaScript Object Notation]], the string value <code>"a\u263Ac"</code>, containing the escape sequence <code>\u263A</code>, would be parsed into <code>"a☺c"</code>.

;Task

Write a function (or routine, subroutine, procedure, etc.) that, when given a [[wp:UTF-8|UTF-8]] encoded string, returns a new string, also encoded in UTF-8, with all escape sequences replaced with equivalent Unicode code points.

For this task we'll follow [[wp:JSON|JSON]] string semantics, defined in [https://datatracker.ietf.org/doc/html/rfc8259#section-7 Section 7] of [https://datatracker.ietf.org/doc/html/rfc8259 RFC 8259]. In summary:

* Allowed two-character escapes are <code>\\</code>, <code>\"</code>, <code>\/</code>, <code>\b</code>, <code>\f</code>, <code>\n</code>, <code>\r</code>, <code>\t</code>.
* <code>"</code>, <code>\</code> and control characters (U+0000 through U+001F) must be escaped.
* Any "character" in the [[wp:Plane_(Unicode)#Basic_Multilingual_Plane|Basic Multilingual Plane]] may be escaped with a <code>\uXXXX</code> sequence, where <code>XXXX</code> are hexadecimal digits encoding the character's code point
* Characters outside the Basic Multilingual Plane may be escaped with a <code>\uXXXX\uXXXX</code> sequence, where <code>XXXX</code> are hexadecimal digits encoding the code point using [[wp:UTF-16|UTF-16]].

Although not required by RFC 8259, we'll treat strings containing invalid <code>\u</code> escape sequences (those that can't be decoded to a Unicode code point) as an error condition.

Test your program with the following inputs and display the string before and after unescaping. It's OK to display bytes rather than "characters" if it's more convenient for your language, or both, if you prefer.
